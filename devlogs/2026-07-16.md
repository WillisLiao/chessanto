# 2026-07-16

## M1: skeleton, board, and game replay

Built the project from scratch per `PLAN.md`:

- XcodeGen (`project.yml`) generates `Chessanto.xcodeproj` from six local
  SPM packages (ChessCore, EngineKit, AnalysisKit, CoachKit, ChessComKit,
  Persistence) plus the SwiftUI app target.
- Forked a background agent to implement `ChessCore` and `EngineKit`
  against the real `chesskit-swift`/`chesskit-engine` source (cloned and
  read directly rather than trusted from summaries) while the main thread
  built Persistence (GRDB schema/store), ChessComKit (public API client),
  and the SwiftUI shell (sidebar, PGN import via file/drag-drop, board
  view, move list, replay stepping) in parallel.
- Real dependency discoveries: `chesskit-engine` runs Stockfish in-process
  via async/await (no subprocess/pipe management needed, simpler than the
  plan originally assumed), has an embedded default NNUE net (no bundling
  required for M1/M2), and its own Stockfish tests are disabled upstream
  as of Stockfish 17 - so EngineKit's tests are intentionally structural
  for now.
- Found and fixed a real bug during E2E verification: `PieceView` used a
  `.font(.system(size: 1000)).minimumScaleFactor(0.01)` hack to size
  Unicode chess glyphs, which produced degenerate/duplicate accessibility
  geometry (all pieces on a rank collapsing to identical reported
  positions). Replaced with GeometryReader-driven font sizing tied to
  actual square size.
- Added accessibility labels to move-list and step-control buttons
  (previously exposed as unlabeled `AXButton`s with no title - a real
  VoiceOver gap, not just a testing inconvenience).

## E2E verification

Fetched a real, current game from the chess.com public API
(`magnuscarlsen`, 2026-07 archive) with clock comments and both castling
sides, and drove the actual built `.app` through AppleScript/System Events:

- Imported the PGN through the real file-import UI (Cmd+O -> Go to Folder
  -> file path) - window title updated to "MagnusCarlsen vs artin10862".
- Quit and relaunched the app; the game was still in the sidebar - GRDB
  persistence confirmed.
- Clicked the real "next move" button four times via `AXPress`; confirmed
  each click by watching the "previous move" button's enabled state flip
  from `false` to `true` after the first click (a clean, unambiguous
  signal that avoided the earlier piece-position confusion).
- Added the real chess.com PGN as a permanent ChessCore test fixture
  (`RealGameFixtureTests.swift`), including a `next(after:)` traversal
  test, since existing tests hadn't exercised that path. The final FEN
  was cross-checked against chess.com's own `[CurrentPosition]` tag for
  the game as independent ground truth.

All package tests pass (`swift test` in each of the six packages) and the
full app test suite passes via `xcodebuild test`.
